## Endpoint

Transform and edit existing images using natural language instructions.

```
POST /v1/images/edits
```

## Authentication

<ParamField header="X-API-Key" type="string" required>
  Your API key for authentication
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Request body

<ParamField body="model" type="string" required>
  The AI model to use for image editing.
  
  **Available models**:
  - `nunchaku-flux.2-klein-4b-base-edit` - Supports tiers: normal, fast, faster
  - `nunchaku-flux.2-klein-4b-edit` - Supports tiers: normal, fast
  - `nunchaku-flux.2-klein-9b-base-edit` - Supports tiers: normal, fast, faster
  - `nunchaku-flux.2-klein-9b-edit` - Supports tiers: normal, fast
  - `nunchaku-qwen-image-edit` - Supports tiers: normal, fast, faster, radically_fast
  - `nunchaku-qwen-image-edit-2509` - Enhanced 2509 variant, supports tiers: normal, fast, faster, radically_fast
  
  See [Models](/concepts/models) for detailed comparison.
</ParamField>

<ParamField body="prompt" type="string" required>
  A text description of the desired edits or transformations.
  
  **Examples**:
  - `"add a sun in the sky"`
  - `"change the background to a beach scene"`
  - `"make the colors more vibrant and add dramatic lighting"`
  - `"remove the person and fill in the background"`
</ParamField>

<ParamField body="url" type="string" required>
  The input image to edit. Can be provided as:
  
  **Data URI** (recommended):
  ```
  data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...
  ```
  
  **HTTP(S) URL**:
  ```
  https://example.com/input-image.png
  ```
  
  **Supported formats**: PNG, JPEG, WebP
  
  **Maximum size**: 10MB
</ParamField>

<ParamField body="tier" type="string" default="fast">
  Performance tier controlling speed/quality tradeoff.
  
  **Options**: `radically_fast`, `faster`, `fast`, `normal`
  
  See [Performance Tiers](/concepts/tiers) for details.
</ParamField>

<ParamField body="size" type="string" default="1024x1024">
  Output image dimensions in format `widthxheight`.
  
  **Common sizes**: `1024x1024`, `1024x768`, `768x1024`, `1280x720`, `720x1280`
</ParamField>

<ParamField body="n" type="integer" default="1">
  Number of edited images to generate (1-10).
</ParamField>

<ParamField body="response_format" type="string" default="url">
  Format for returned images.
  
  **Options**:
  - `url` - Returns a URL to the hosted image
  - `b64_json` - Returns base64-encoded image data
</ParamField>

<ParamField body="seed" type="integer">
  Random seed for reproducible editing. Use the same seed with identical parameters for consistent results.
</ParamField>

## Response

<ResponseField name="created" type="integer">
  Unix timestamp of when the image was edited
</ResponseField>

<ResponseField name="data" type="array">
  Array of edited images
  
  <Expandable title="data item properties">
    <ResponseField name="url" type="string">
      URL to the edited image (when `response_format: "url"`)
    </ResponseField>
    
    <ResponseField name="b64_json" type="string">
      Base64-encoded image data (when `response_format: "b64_json"`)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="model" type="string">
  The model that was used for editing
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
# First, base64-encode your input image:
# base64 -w 0 input.png > input_b64.txt

curl -X POST http://74.81.65.159:30371/v1/images/edits \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "model": "nunchaku-flux.2-klein-9b-edit",
    "tier": "normal",
    "prompt": "add a sun in the sky",
    "url": "data:image/png;base64,<BASE64_IMAGE_DATA>",
    "n": 1,
    "size": "1024x1024",
    "response_format": "b64_json",
    "seed": 42
  }'
```

```python Python (Requests)
import base64
from pathlib import Path

import requests

url = "http://74.81.65.159:30371/v1/images/edits"
headers = {
    "X-API-Key": "YOUR_API_KEY",
    "Content-Type": "application/json",
}

# Read and encode the input image
img_b64 = base64.b64encode(Path("input.png").read_bytes()).decode()

payload = {
    "model": "nunchaku-flux.2-klein-9b-edit",
    "tier": "normal",
    "prompt": "add a sun in the sky",
    "url": f"data:image/png;base64,{img_b64}",
    "n": 1,
    "size": "1024x1024",
    "response_format": "b64_json",
    "seed": 42,
}

response = requests.post(url, json=payload, headers=headers)
response.raise_for_status()

result = response.json()
image_b64 = result["data"][0]["b64_json"]
Path("edited.png").write_bytes(base64.b64decode(image_b64))
```

```python Python (OpenAI SDK)
import base64
from pathlib import Path

from openai import OpenAI

# pip install openai
client = OpenAI(
    base_url="http://74.81.65.159:30371/v1",
    api_key="YOUR_API_KEY",
)

# Encode local image as data URL
img_b64 = base64.b64encode(Path("input.png").read_bytes()).decode()
data_url = f"data:image/png;base64,{img_b64}"

response = client.chat.completions.create(
    model="nunchaku-flux.2-klein-9b-edit",
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "add a sun in the sky"},
                {"type": "image_url", "image_url": {"url": data_url}},
            ],
        },
    ],
    extra_body={
        "tier": "normal",
        "seed": 42,
    },
)

# Extract edited image from response
content = response.choices[0].message.content
image_url = content[0]["image_url"]["url"]
_, b64_data = image_url.split(",", 1)
Path("edited.png").write_bytes(base64.b64decode(b64_data))
```


</CodeGroup>

## Best practices

<Tip>
  **Effective editing prompts**:
  - Be specific about what to change, add, or remove
  - Describe the desired outcome clearly
  - Mention style, colors, and atmosphere when relevant
  - For complex edits, break into multiple sequential requests
</Tip>

<Warning>
  **Image preparation tips**:
  - Use high-quality input images for best results
  - Ensure images are under 10MB
  - Supported formats: PNG, JPEG, WebP
  - Base64 encoding increases payload size
</Warning>

<Note>
  **Performance considerations**:
  - Qwen models provide highest quality edits
  - FLUX Klein models are faster for simpler edits
  - Use `tier: "faster"` for quick iterations
  - Use `tier: "normal"` for final, high-quality outputs
</Note>

```bash Bash
    \"model\": \"nunchaku-qwen-image-edit\",
    \"prompt\": \"add a sun in the sky\",
    \"url\": \"data:image/png;base64,${IMG_B64}\"
  }"
```

```python Python
import base64

# Encode image
with open("input.png", "rb") as f:
    img_b64 = base64.b64encode(f.read()).decode()

# Create data URI
data_uri = f"data:image/png;base64,{img_b64}"
```
