## Endpoint

Generate images from text descriptions using AI models.

```
POST /v1/images/generations
```

## Authentication

<ParamField header="X-API-Key" type="string" required>
  Your API key for authentication
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Request body

<ParamField body="model" type="string" required>
  The AI model to use for generation.
  
  **Available models**:
  - `nunchaku-flux.2-klein-4b-base` - Supports tiers: normal, fast, faster
  - `nunchaku-flux.2-klein-4b` - Supports tiers: normal, fast
  - `nunchaku-flux.2-klein-9b-base` - Supports tiers: normal, fast, faster
  - `nunchaku-flux.2-klein-9b` - Supports tiers: normal, fast
  - `nunchaku-qwen-image` - Supports tiers: normal, fast, faster, radically_fast
  - `nunchaku-z-image-turbo` - Distilled model (no tier field needed)
  
  See [Models](/concepts/models) for detailed comparison.
</ParamField>

<ParamField body="prompt" type="string" required>
  A text description of the desired image. Be specific and descriptive for best results.
  
  **Example**: `"professional product photo of red wireless headphones on white background with soft studio lighting"`
</ParamField>

<ParamField body="tier" type="string" default="fast">
  Performance tier controlling speed/quality tradeoff.
  
  **Options**: `radically_fast`, `faster`, `fast`, `normal`
  
  See [Performance Tiers](/concepts/tiers) for details.
</ParamField>

<ParamField body="size" type="string" default="1024x1024">
  Image dimensions in format `widthxheight`.
  
  **Common sizes**: `1024x1024`, `1024x768`, `768x1024`, `1280x720`, `720x1280`
</ParamField>

<ParamField body="n" type="integer" default="1">
  Number of images to generate (1-10).
</ParamField>

<ParamField body="response_format" type="string" default="url">
  Format for returned images.
  
  **Options**:
  - `url` - Returns a URL to the hosted image
  - `b64_json` - Returns base64-encoded image data
</ParamField>

<ParamField body="seed" type="integer">
  Random seed for reproducible generation. Use the same seed with identical parameters for consistent results.
</ParamField>

## Response

<ResponseField name="created" type="integer">
  Unix timestamp of when the image was generated
</ResponseField>

<ResponseField name="data" type="array">
  Array of generated images
  
  <Expandable title="data item properties">
    <ResponseField name="url" type="string">
      URL to the generated image (when `response_format: "url"`)
    </ResponseField>
    
    <ResponseField name="b64_json" type="string">
      Base64-encoded image data (when `response_format: "b64_json"`)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="model" type="string">
  The model that was used for generation
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl -X POST http://74.81.65.159:30371/v1/images/generations \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "model": "nunchaku-flux.2-klein-9b",
    "tier": "normal",
    "prompt": "A beautiful sunset over mountains",
    "n": 1,
    "size": "1024x1024",
    "response_format": "b64_json",
    "seed": 42
  }'
```

```python Python (Requests)
import base64
from pathlib import Path

import requests

url = "http://74.81.65.159:30371/v1/images/generations"
headers = {
    "X-API-Key": "YOUR_API_KEY",
    "Content-Type": "application/json",
}

payload = {
    "model": "nunchaku-flux.2-klein-9b",
    "tier": "normal",
    "prompt": "A beautiful sunset over mountains",
    "n": 1,
    "size": "1024x1024",
    "response_format": "b64_json",
    "seed": 42,
}

response = requests.post(url, json=payload, headers=headers)
response.raise_for_status()

result = response.json()
image_b64 = result["data"][0]["b64_json"]
Path("output.png").write_bytes(base64.b64decode(image_b64))
```

```python Python (OpenAI SDK)
import base64
from pathlib import Path

from openai import OpenAI

# pip install openai
client = OpenAI(
    base_url="http://74.81.65.159:30371/v1",
    api_key="YOUR_API_KEY",
)

response = client.images.generate(
    model="nunchaku-flux.2-klein-9b",
    prompt="A beautiful sunset over mountains",
    size="1024x1024",
    response_format="b64_json",
    n=1,
    extra_body={
        "tier": "normal",
        "seed": 42,
    },
)

image_b64 = response.data[0].b64_json
Path("output.png").write_bytes(base64.b64decode(image_b64))
```

### Saving base64 image to file

```bash
# Using jq and base64 to decode and save
curl -s -X POST http://74.81.65.159:30371/v1/images/generations \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "model": "nunchaku-flux.2-klein-9b",
    "tier": "normal",
    "prompt": "A beautiful sunset over mountains",
    "response_format": "b64_json"
  }' | jq -r '.data[0].b64_json' | base64 -d > output.png
```

## Best practices

<Tip>
  **Prompt engineering tips**:
  - Be specific about style, lighting, and composition
  - Include relevant details like colors, materials, atmosphere
  - Mention artistic style or reference when relevant
  - Use natural, descriptive language
</Tip>

<Warning>
  **Performance considerations**:
  - Start with `tier: "fast"` for balanced performance
  - Use `radically_fast` or `faster` for real-time applications
  - Use `normal` only when maximum quality is essential
  - Larger images and multiple generations increase processing time
</Warning>
