## Getting started

This page provides practical examples for generating images from text descriptions using different models, tiers, and configurations.

## Basic generation

### Simple image generation

The most basic request to generate an image:

```bash
curl -X POST http://74.81.65.159:30371/v1/images/generations \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "nunchaku-qwen-image",
    "prompt": "a red apple on a wooden table"
  }'
```

This uses default values:
- `tier`: `fast`
- `size`: `1024x1024`
- `n`: `1`
- `response_format`: `url`

### With all parameters specified

```bash
curl -X POST http://74.81.65.159:30371/v1/images/generations \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "nunchaku-qwen-image",
    "tier": "fast",
    "prompt": "professional product photo of red wireless headphones on white background with soft studio lighting",
    "size": "1024x1024",
    "n": 1,
    "response_format": "b64_json",
    "seed": 42
  }' | jq -r '.data[0].b64_json' | base64 -d > output.png
```

## Batch generation

### Generate multiple variations

```bash
#!/bin/bash

PROMPT="abstract art with vibrant colors and dynamic composition"

# Generate 5 variations with different seeds
for seed in 42 123 456 789 1024; do
  echo "Generating variation with seed ${seed}..."
  curl -s -X POST http://74.81.65.159:30371/v1/images/generations \
    -H "X-API-Key: YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d "{
      \"model\": \"nunchaku-qwen-image\",
      \"tier\": \"fast\",
      \"prompt\": \"${PROMPT}\",
      \"size\": \"1024x1024\",
      \"response_format\": \"b64_json\",
      \"seed\": ${seed}
    }" | jq -r '.data[0].b64_json' | base64 -d > "variation_${seed}.png"
  echo "Saved variation_${seed}.png"
done
```

### Generate with multiple prompts

```bash
#!/bin/bash

# Array of prompts
prompts=(
  "a red apple on a wooden table"
  "a green pear on a marble counter"
  "a yellow banana on a ceramic plate"
  "an orange on a rustic wooden board"
)

# Generate images for each prompt
for i in "${!prompts[@]}"; do
  echo "Generating image $((i+1))/${#prompts[@]}..."
  curl -s -X POST http://74.81.65.159:30371/v1/images/generations \
    -H "X-API-Key: YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d "{
      \"model\": \"nunchaku-flux.2-klein-9b\",
      \"tier\": \"faster\",
      \"prompt\": \"${prompts[$i]}\",
      \"response_format\": \"b64_json\"
    }" | jq -r '.data[0].b64_json' | base64 -d > "fruit_${i}.png"
done
```

## Programming language examples

<CodeGroup>

```python Python
import requests
import base64
import os

def generate_image(prompt, model="nunchaku-qwen-image", tier="fast", size="1024x1024", seed=None):
    """Generate an image and save it to a file."""
    payload = {
        "model": model,
        "tier": tier,
        "prompt": prompt,
        "size": size,
        "response_format": "b64_json"
    }
    
    if seed is not None:
        payload["seed"] = seed
    
    response = requests.post(
        "http://74.81.65.159:30371/v1/images/generations",
        headers={
            "X-API-Key": os.environ["NUNCHAKU_API_KEY"],
            "Content-Type": "application/json"
        },
        json=payload
    )
    
    result = response.json()
    image_b64 = result["data"][0]["b64_json"]
    
    # Decode and return image data
    return base64.b64decode(image_b64)

# Generate multiple variations
prompts = [
    "a red apple on a wooden table",
    "a modern office workspace",
    "abstract colorful background"
]

for i, prompt in enumerate(prompts):
    print(f"Generating image {i+1}/{len(prompts)}...")
    image_data = generate_image(prompt, seed=42+i)
    
    with open(f"output_{i}.png", "wb") as f:
        f.write(image_data)
    
    print(f"Saved output_{i}.png")
```

```javascript JavaScript
import fetch from 'node-fetch';
import fs from 'fs';

async function generateImage(prompt, options = {}) {
  const {
    model = 'nunchaku-qwen-image',
    tier = 'fast',
    size = '1024x1024',
    seed = null
  } = options;
  
  const payload = {
    model,
    tier,
    prompt,
    size,
    response_format: 'b64_json'
  };
  
  if (seed !== null) {
    payload.seed = seed;
  }
  
  const response = await fetch(
    'http://74.81.65.159:30371/v1/images/generations',
    {
      method: 'POST',
      headers: {
        'X-API-Key': process.env.NUNCHAKU_API_KEY,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(payload)
    }
  );
  
  const result = await response.json();
  const imageB64 = result.data[0].b64_json;
  
  return Buffer.from(imageB64, 'base64');
}

// Generate multiple variations
const prompts = [
  'a red apple on a wooden table',
  'a modern office workspace',
  'abstract colorful background'
];

for (let i = 0; i < prompts.length; i++) {
  console.log(`Generating image ${i+1}/${prompts.length}...`);
  
  const imageBuffer = await generateImage(prompts[i], { seed: 42 + i });
  fs.writeFileSync(`output_${i}.png`, imageBuffer);
  
  console.log(`Saved output_${i}.png`);
}
```

Use the same seed for consistent results:

```bash
# These will generate similar (not identical) images
curl -X POST http://74.81.65.159:30371/v1/images/generations \

  -H "Content-Type: application/json" \
  -d '{
    "model": "nunchaku-qwen-image",
    "prompt": "a sunset over mountains",
    "seed": 12345

curl -X POST http://74.81.65.159:30371/v1/images/generations \
```

## Best practices

<AccordionGroup>
  <Accordion title="Prompt engineering" icon="pen">
    - Be specific and descriptive
    - Include style, lighting, and composition details
    - Mention materials, colors, and atmosphere
    - Reference artistic styles when relevant
    - Use natural language, avoid keyword stuffing
  </Accordion>
  
  <Accordion title="Model selection" icon="layer-group">
    - Use Qwen Image for highest quality and complex prompts
    - Use FLUX Klein 9B for balanced performance
    - Use FLUX Klein 4B or Z-Image Turbo for speed
    - Consider your quality vs. speed requirements
  </Accordion>
  
  <Accordion title="Performance optimization" icon="gauge">
    - Start with `tier: "fast"` for most use cases
    - Use `faster` or `radically_fast` for real-time apps
    - Use `normal` only when maximum quality is essential
    - Batch requests when generating multiple images
  </Accordion>
</AccordionGroup>
